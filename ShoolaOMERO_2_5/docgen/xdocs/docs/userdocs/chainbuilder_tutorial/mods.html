OME's module palette presents a graphical display of widgets representing the modules in an OME configuration. The modules are arranged by category, with each category taking the form of a bordered box that surrounds all modules in that category. Subcategories are contained within categories, alongside of modules.

Each module is initially shown in its "overview" mode. in this view, the module is shown simply with its name and a summary representation of all inputs and ouputs.

 
Fig 2-1: The Chain module palette.
Tip
When the chain builder starts, the module palette is intentionally drawn very small, in order to conserve screen space. You'll almost certainly want to move and resize it to suit your needs.

There are two strategies that can be used in combination to see more details about individual modules:

  • Zooming:Clicking on a module will cause the display to zoom to be centered around that module. Similarly, clicking in a category box (on its border or anywhere inside the box that is not also inside a module) will zoom the display to focus on a category box. Clicking on the background, or right-clicking on a module/category box, will cause the display to zoom out to a higher-level overview.
  • Mouse-over: Mousing over a module will make it expand, showing detailed descriptions of inputs and outputs.

These techniques can, of course, be used in combination.

 
Fig 2-2: The Chain module palette, after zooming into a module category and mousing over a module.

The module library can also be navigated via a tree view. To access the tree view, click on the right arrow on the bottom-left corner of the window. This will open a sub-window containing a categorized list of modules, using a tree view similar to those used in file-system browsers. This view is coordinated with the palette view: selecting a module or category in the tree will cause it to be highlighted in the palette. The amount of space given to the tree view can be adjusted by dragging the bar that separates the tree view from the palette view. To hide the tree view, click the left-arrow at the bottom of the separating bar.

 
Fig 2-3: The Chain module palette, with the tree.

The module palette provides interactive feedback regarding possible connections between modules. When the mouse is moved over a module, all modules that can link to that module (i.e., those that have inputs/outputs with semantic types that match a corresponding output/input on the selected module) are highlighted in dark blue. This highlighting provides some guidance that might be useful in selecting modules for chain creation. This higlighting will be discussed in more detail in the chapter on chain creation.
