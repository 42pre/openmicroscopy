Shoola is entirely written in Java and you will need a Java Runtime Environment (version 1.4 or later) to run it. All the third-party libraries required by Shoola come already in each distribution bundle and are automatically linked at runtime, so you won't have to worry about installing/upgrading libraries.

For the sake of record, here's a brief description of the third-party libraries used by Shoola. First off, all of those libraries are stored in the LIB directory under the CVS root. Its test sub-directory contains additional libraries required by the test code — these are not obviously included in the distribution bundles. The libraries required by the Shoola source code are:

  • piccolo.jar and piccolox.jar: The Piccolo graphics framework (http://www.cs.umd.edu/hcil/jazz/). Version 1.0. These libraries are used by Shoola to build zoomable user interfaces.
  • log4j-1.2.8.jar: The Log4j logging framework (http://logging.apache.org/log4j/). Version 1.2.8. This library provides logging facilities to Shoola.
  • ome-java.jar: This library is part of the OME distributions and is used by Shoola as a gateway to the OME data and image servers.
  • xmlrpc-1.2-b1.jar: The Apache XmlRpc package (http://ws.apache.org/xmlrpc/). Version 1.2-b1. This library is not directly required by Shoola, but by the ome-java.jar, which uses it to connect to the OME data server.
  • commons-httpclient-2.0-rc2.jar: The The Jakarta Commons HttpClient (http://jakarta.apache.org/commons/httpclient/). Version 2.0-rc2. This library is not directly required by Shoola, but by the ome-java.jar, which uses it to connect to the OME image server.
  • commons-logging.jar: The Jakarta Commons Logging (http://jakarta.apache.org/commons/logging/). Version 1.0. This library is not directly required by Shoola, but by the commons-httpclient-2.0-rc2.jar.

 

The additional libraries required by the test code are:

  • junit-3.8.1.jar: The JUnit Testing Framework (http://www.junit.org/). Version: 3.8.1.
